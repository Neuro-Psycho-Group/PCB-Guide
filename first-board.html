08 / First Board

Your First PCB

A complete walkthrough: design a real board in KiCad, generate Gerbers, and order it. Every single step, nothing assumed.

The project

What We're Building

An LED blinker: an ATtiny85 microcontroller that blinks three LEDs in a pattern. Simple enough to finish in an evening, complex enough to teach every step of the design process — schematic, layout, routing, Gerber export, and ordering.

Why this specific project?

This board is perfectly calibrated for a first build. It's simple enough that you can finish it in one session, but it covers everything: a power circuit, passive components, an IC with decoupling, GPIO outputs with calculated resistors, a user input, and a programming header. Every subsequent board you design will use these same building blocks.

LED Resistor Calculation

Never connect an LED directly to a microcontroller pin without a resistor — it will burn out instantly. The resistor limits current to a safe level.

// For a red/yellow LED on 5V supply, 10mA target current

Supply voltage (Vsupply)=5V
LED forward voltage (Vf)=2.0V (red/yellow) · 3.0V (blue/white)
Target current (If)=10mA = 0.010A
R = (Vsupply − Vf) / If=(5 − 2.0) / 0.010 = 300Ω → use 330Ω or 470Ω (nearest E24 values)
// For blue/white LEDs: (5 − 3.0) / 0.010 = 200Ω → use 220Ω

Step by step

Building It in KiCad

These steps assume KiCad 7 or 8. If you haven't installed it yet — download it free from kicad.org. The interface looks complex at first; don't let that put you off. You'll only use about 10% of it for this project.

1

Create a new project

~2 min
  1. Open KiCad. Click File → New Project
  2. Create a folder called attiny85-blinker and save the project inside it
  3. KiCad creates two files: .kicad_sch (schematic) and .kicad_pcb (layout). You'll work in both.
KiCad tip: Save often. Ctrl+S works everywhere in KiCad. The tool does not autosave by default.
2

Draw the schematic

~30 min

Double-click the .kicad_sch file to open the Schematic Editor. This is where you place component symbols and draw connections.

  1. Press A to open the component picker. Search for ATtiny85 and place it in the centre of the canvas.
  2. Search and place all other components: LED, R (resistor), C (capacitor), USB_B_Micro, SW_Push, Conn_02x03_Odd_Even (ISP header).
  3. Press W to draw wires. Connect pins as per the schematic above.
  4. Add power symbols: press P, search +5V for the positive rail and GND for ground. Place these on every VCC and GND pin.
  5. Add component values: double-click each component and fill in the Value field (470, 10k, 100n, etc.)
KiCad tip: Press E on any component to edit its properties. Press R to rotate before placing. Press G while hovering a component to grab and move it with wires attached.
Why power symbols? Instead of drawing a wire from +5V all the way across the schematic to every VCC pin, you place a +5V symbol on each pin. KiCad knows that all +5V symbols are connected — cleaner schematic, same electrical result.
  1. Run the Electrical Rules Check: Inspect → Electrical Rules Checker → Run ERC. Fix all errors before proceeding. Common first-timer issues: missing power pins, unconnected component pins.
3

Assign footprints

~15 min

Open Tools → Assign Footprints. For each component, select a physical footprint from KiCad's built-in library.

4

Set board outline and place components

~20 min

Open the PCB Editor (Tools → Open PCB in Editor or click the PCB file). Import the schematic netlist: Tools → Update PCB from Schematic. All your components appear in a pile. Now arrange them.

  1. Draw the board outline: switch to the Edge.Cuts layer, press R, draw a 40×30mm rectangle. This defines your physical board boundary.
  2. Drag components inside the outline. Suggested placement:
    • USB connector: bottom edge, centred
    • ISP header: top-right corner
    • ATtiny85: centre of board
    • Reset button: top-left area
    • LEDs: spread across the top
    • Resistors and capacitors: close to the components they connect to
  3. Place C1 (100nF decoupling cap) physically as close to the ATtiny85's VCC pin as possible — ideally within 2mm.
Why placement matters so much: Good placement means short traces, which means less resistance, less noise, and easier routing. The ratsnest lines (thin grey lines showing unrouted connections) should look tidy and un-crossed after good placement. If they look like a tangled mess, rearrange components until they don't.
5

Route traces

~25 min

Your goal: replace every ratsnest line with a routed copper trace. No thin lines remaining = fully routed.

Set your trace widths before starting. Go to Board Setup → Design Rules → Net Classes:

  • Default net class: 0.25mm trace width, 0.25mm clearance
  • Create a "Power" net class, assign +5V and GND: 0.5mm trace width

Now route traces. Press X to start routing. Click a pad to begin, route to the other pad, click to finish. Follow these routing rules:

  1. Route power first: +5V from USB → C2 → C1 → ATtiny85 VCC, then ISP header VCC
  2. Route GND: use the bottom copper layer for a ground plane (see step 6)
  3. Route ATtiny85 PB0/PB1/PB2 to LED resistors, then to LED anodes
  4. Route LED cathodes to GND
  5. Route reset button, pull-up resistor, and ISP header RST pin
KiCad tip: Press / while routing to switch between top copper (F.Cu) and bottom copper (B.Cu) — KiCad automatically adds a via at the layer transition. Use this to avoid crossing traces.

Don't panic about perfection

Your first board will not be perfectly routed. Traces will be longer than ideal, some will take detours. That's fine — this is a learning exercise. As long as the DRC passes and all nets are connected, the board will work.

6

Add ground plane and finishing touches

~10 min
  1. Select the B.Cu (bottom copper) layer. Press Z to add a filled zone. Draw a rectangle covering the entire board area inside the Edge.Cuts boundary. Set the zone net to GND. Press B to fill all zones.
  2. The bottom copper layer now floods with GND copper everywhere it can fit. This is your ground plane.
  3. Add silkscreen labels: switch to F.Silkscreen, add text for board name, revision, and any labels you want (+ polarity on C2, etc.)
  4. Add mounting holes if desired: Add Footprint → search MountingHole → place in corners
Why a ground plane? Instead of routing individual GND traces to every component, a copper pour on the bottom layer connects all GND pads automatically (via their connection to the flood). Lower impedance, less noise, and your routing becomes much simpler — any GND pad is automatically connected by the nearest via to the pour.
7

Run DRC and generate Gerbers

~10 min
  1. Run Inspect → Design Rules Checker → Run DRC. Fix every error. Common issues: traces too close to board edge, unconnected nets (you missed routing something), clearance violations.
  2. Go to File → Fabrication Outputs → Gerbers. Make sure all layers are ticked: F.Cu, B.Cu, F.Mask, B.Mask, F.Silkscreen, Edge.Cuts. Click Generate.
  3. Also generate: File → Fabrication Outputs → Drill Files. Use Excellon format.
  4. Put all generated files in a zip archive.
  5. Preview the Gerbers by uploading the zip to JLCPCB's Gerber viewer. Verify each layer looks correct before ordering.
8

Order and wait

~5 min + shipping
  1. Go to jlcpcb.com. Click "Order Now".
  2. Upload your Gerber zip. Their system auto-detects layers, board size, and drill count.
  3. Settings: 5 boards · 1.6mm FR4 · Green solder mask · HASL lead-free · 1oz copper. Leave everything else as default.
  4. Add to cart, checkout. Shipping to UK via DHL: approximately 7–14 days.
  5. While you wait: write the ATtiny85 firmware. A simple blink sketch in the Arduino IDE with ATtiny85 board support takes about 10 minutes.

Programming the ATtiny85

The ATtiny85 is programmed via its ISP header (J2 on your board) using a programmer like a USBasp or another Arduino acting as "Arduino as ISP". Install ATtinyCore in the Arduino IDE board manager, select ATtiny85, set clock to 8MHz internal, and upload. Your ISP header pinout should follow the standard AVRISP/USBasp 6-pin layout.

After your first board

What to Build Next

You've now done everything once. Every subsequent board uses the same skills — just more of them, and more complex. Here's a suggested progression.

Project 2

SMD LED blinker

Rebuild the same board using 0805 SMD components instead of through-hole. Same circuit, different soldering challenge. Teaches SMD footprints and soldering technique.

Project 3

STM32 or RP2040 breakout

A 32-bit microcontroller breakout with USB, decoupling, crystal, and GPIO headers. Introduces multi-layer concepts and more complex IC packages (QFP/QFN).

Project 4

Custom power supply

A buck converter (e.g. LM2596 or MP2307) stepping 12V down to 5V. Introduces power design, inductor selection, and thermal management.

Project 5

Sensor + display board

I²C sensor (BME280 temperature/humidity) feeding an OLED display via a microcontroller. Introduces I²C routing, pull-up resistors, and display interfaces.

Project 6

Motor driver

An H-bridge (DRV8833 or L298N) controlling DC motors with PWM. Teaches high-current trace sizing, flyback diodes, and mixed power/signal design.

Long-term

4-layer design

When a 2-layer board starts to feel cramped or noisy, move to 4 layers: dedicated ground and power planes sandwiched inside. Transforms signal integrity.
